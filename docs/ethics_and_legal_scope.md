# Ethics and Legal Scope

## Authorized Local Lab Only

This project is designed for a controlled academic lab. It must only be run against:

- localhost,
- Docker Compose services in this repository,
- synthetic logs generated by the simulator,
- safe public datasets if a future version adds dataset ingestion.

## Explicitly Out of Scope

The project must not be used for:

- scanning external IP addresses or domains,
- brute-forcing real accounts,
- exploiting public systems,
- collecting real personal data,
- credential theft,
- malware execution,
- data exfiltration,
- unauthorized penetration testing.

## Synthetic Data

The simulator generates structured event records. Suspicious web activity is represented with harmless markers such as `SIMULATED_WEB_ATTACK_MARKER`. These markers are not real exploit payloads and are only used to test defensive detection logic.

## AI Use

The Random Forest model is used for defensive classification and education. Model limitations must be disclosed clearly:

- training data is synthetic,
- metrics represent controlled lab performance,
- false positives and false negatives are possible,
- recommendations are educational and not a substitute for professional incident response.

## Responsible Reporting

Evidence exports are intended for the final report and presentation. They should not include real credentials, real user data, or logs from systems outside the authorized lab boundary.
